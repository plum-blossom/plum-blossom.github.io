一、基础环境配置
1.安装node
node：一款运行效率非常高的web服务器，采用JavaScript语法
注意：建议安装版本6.x
测试环境：
win+r回车，输入cmd回车，输入node -v
2.安装git
git：版本控制器，团队协作开发的时候使用
二、安装hexo脚手架
npm install hexo-cli -g //在全局安装hexo脚手架
从node服务器下载hexo脚手架到本机
node服务器在国外，怎么办？
切换镜像（切换服务器）：npm install -g cnpm –registry=https://registry.npm.taobao.org
安装：
cnpm install hexo-cli -g //在全局安装hexo脚手架
三、创建博客项目
1.桌面新建名为blog的文件夹（Git Bash Here ）
命令：mkdir blog
2.进入blog文件夹
命令：cd blog
3.初始化博客项目（在blog文件夹完成）
命令：hexo init
4.导入npm依赖包
命令：cnpm install （简写：cnpm i）
5.启动hexo项目服务器
命令：hexo s
启动之后可以在浏览器地址输入：localhost：4000预览博客
如若4000端口浏览不出，用下命令：
hexo server -p 4001
6.编译博客
每当博客内容有更改的时候，需要先编译（Visual Studio）再刷新浏览器
命令：hexo g
添加文章命令：hexo new 文章名
四、美化博客
1.更换博客主题
cd themes
git clone https://github.com/WongMinHo/hexo-theme-miho.git
修改hexo根目录下的 _config.yml ： theme: hexo-theme-miho
保存然后hexo g
2.另一个主题：
$ hexo init blog
$ cd blog && npm install
$ git clone https://github.com/littlewin-wang/hexo-theme-casual themes/casual
修改hexo根目录下的 _config.yml ： theme: casual
3.如果不行先npm install –save hexo-renderer-jade hexo-generator-feed hexo-generator-sitemap hexo-browsersync hexo-generator-archive，在git clone。
4.导航条设置
命令：
hexo new page 关于
5.主题
git clone https://github.com/shenliyang/hexo-theme-snippet.git
四、git常用命令
1.在任意位置选择右键选择git bash here可以再cmd打开当前路径
2.路径转换：
返回上级目录：cd ..
进入下级目录：cd 路径名称
3.显示当前路径的文件、夹列表：ls
4.创建文件夹：mkdir 文件名
5.切换盘符：
如：当前在c盘某路径，需要进入d盘某路径

1. d： 回车（输入d冒号空号然后回车）
2. cd 详细路径

五、gitHub的使用
1、本地创建ssh key
桌面右键git bash here,然后输入ssh-keygen -t rsa -C “邮箱”
记住ssh key的保存路径，然后一路回车，直到显示很多小圆圈
默认路径：c:/users/用户名/.ssh
2.打开c:/users/用户名/.ssh文件夹下的id_rsa.pub文件（记事本）
Ctrl+a全选，后Ctrl+c复制备用
3.打开github.com登录账户，点击头像右侧小三角在在下拉菜单中选择settings
在新页面左侧选择ssh and gpg keys，然后点击右侧绿色new ssh key按钮
title：随意填
key：Ctrl+V粘贴刚才复制的key，然后点击下方绿色add ssh key按钮
4.回到git命令行工具，输入ssh -T git@github.com回车 (连接到github服务器)
如果提示Hi plum-blossom! You’ve successfully authenticated, but GitHub does not provide shell access.代表key成功
5.验证github邮箱和用户名：
在git命令行窗口输入：
git config –global user.email “you@example.com”
git config –global user.name “Your Name”
注意：编辑英文符号和空格
6.github创建仓库
github页面点击头像左侧加号，点击下拉列表第一个按钮
填写Repository name，仓库名，后点击最下方Create repository按钮

#7.桌面右键git bash here，copy以下命令：
  echo "#test" >> README.md   #在桌面创建内容为“测试”的md文件
  git init                    #初始化git

  git add README.md           #添加刚才生成的md文件

  git commit -m "我的第一次提交"   #编写更新说明
  git remote add origin https://github.com/plum-blossom/test.git  #设置需要提交的仓库

  git push -u origin master  #向仓库推送文件
   第一次推送成功后，后续还需要推送文件到之前仓库怎么办？三步：
    git add 文件名.后缀
    Git commit -m "更新说明"
    git push -u origin master
----------------------------------------------------------------------------------------------------|

！注意： #git add “目录”
git add .都会提示这样一个警告消息：
warning: LF will be replaced by CRLF in XXXXXXXXXXXXXX.
虽然说没有什么影响吧。不过就是觉得太碍眼了，按照这样设置就没有问题了:

git config core.autocrlf false

这样设置git的配置后在执行add操作就没有问题了。
—————————————————————————————————–|
！注意：如果输入$ git remote add origin git@github.com:djqiang（github帐号名）/gitdemo（项目名）.git
提示出错信息：fatal: remote origin already exists.
解决办法如下：
1、先输入$ git remote rm origin
2、再输入$ git remote add origin git@github.com:djqiang/gitdemo.git 就不会报错了！
—————————————————————————————————–|
如果输入$ git push origin master
提示出错信息：error:failed to push som refs to …….
解决办法如下：
1、先输入$ git pull origin master //先把远程服务器github上面的文件拉下来
2、再输入$ git push origin master
3、如果出现报错 fatal: Couldn’t find remote ref master或者fatal: ‘origin’ does not appear to be a git repository以及fatal: Could not read from remote repository.
4、则需要重新输入$ git remote add origin git@github.com:djqiang/gitdemo.git

github常见操作和常见错误网址：http://blog.csdn.net/dengjianqiang2011/article/details/9260435

六、代码托管到github
1.在博客项目根目录的_config.yml文件的底部添加如下信息：
deploy：
type: git
repository: https://github.com/plum-blossom/plum-blossom.github.io.git
branch: master
注意格式的缩进跟_config.yml里面其他项保持一致
2,。添加依赖包：
命令执行：npm install hexo-deployer-git –save (在Git bash here中)
3.执行：hexo g编译博客项目
4.执行hexo d将代码托管到github
如果成功，可以通过用户名.github.io访问个人博客
手机访问plum-blossom.github.io
登录github.com在建新存储库，名为plum-blossom.github.io
